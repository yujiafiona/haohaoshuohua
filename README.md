# 好好说话 · AI 沟通助手 MVP

「好好说话」是一个面向日常关系的 **非暴力沟通 / 关键对话 AI 训练助手**，网站版 MVP 适合部署在 Vercel。

> 标语：**让情绪被理解，而不是被放大**

---

## 项目概览

### 背景

- **问题**：日常沟通里容易「一开口就怼」——情绪上头时话一出口就带刺，事后又后悔；想好好说又不知道怎么说才既保留诉求又不伤关系。
- **定位**：不做「吵架话术生成器」，而是做**沟通训练助手**：帮用户把「想怼的话」翻译成更好说出口的版本，并在安全的环境里反复练习，形成习惯。理论依据来自《非暴力沟通》和《关键对话》。
- **目标用户**：在意关系质量、愿意在关键对话前打草稿或事后复盘的人（职场、亲密关系、家庭、朋友等）。

### 功能点

| 功能 | 说明 |
|------|------|
| **情绪翻译** | 输入一句情绪化原话 → 选择沟通对象 + 场景（可选填对方特点）→ 得到「好好说话版」表达 + 怒气值/攻击性/共情度/清晰度评分 + 沟通小贴士。结果可一键复制。 |
| **双筛选** | 沟通对象：老板 / 同事 / 伴侣 / 父母 / 朋友；场景：提需求·求助 / 拒绝·说不 / 安慰·支持 / 表达不满·冲突前。 |
| **角色记忆** | 可选一句话描述对方特点或雷区（如「怕被否定」「时间很紧」），AI 在翻译和出题时会参考。 |
| **沟通训练** | 根据对象+场景+角色出题 → 用户写一句「会发出去的话」→ AI 按【总体】【亮点】【可优化】【关联】分段点评，并支持「再来一题」持续练习。 |
| **功德成长** | 每次成功使用（翻译 / 出题 / 打分 / 再来一题）+5 功德值，本地累计；等级：初识表达 → 情绪译者 → 沟通练习生 → 非暴力玩家 → 关键对话高手。 |
| **演示模式** | 未配置 API Key 时自动返回示例数据，链接可直接发给他人完整体验。 |

### 价值

- **对用户**：在「想说又不敢说 / 一说就崩」的场景里，先在这里把话翻译、练一遍，再在真实对话里用，降低关系风险、提高表达清晰度。
- **对产品**：把「非暴力沟通 / 关键对话」从书本落到可用的轻工具，强调训练与复盘而非一次性话术，便于后续做 RAG 知识库、练习记录、打卡与传播。

## 功能概览

- **情绪翻译**
  - 输入一段情绪化、想怼人的原话
  - 选择沟通对象（老板 / 同事 / 伴侣 / 父母 / 朋友）+ 场景（提需求 / 拒绝 / 安慰 / 表达不满）
  - 可选补充：对方特点 / 雷区（角色记忆）
  - AI 输出：优化后的「好好说话版表达」+ 怒气值 / 攻击性 / 共情度 / 清晰度评分
  - 给出基于《非暴力沟通》《关键对话》的生活化理论建议

- **沟通训练模式**
  - 结合对象 + 场景 + 角色特点自动出题
  - 用户写「会发出去的一句话」
  - AI 从安全感、尊重感、清晰度等维度打分，给出亮点 & 改写建议

- **成长体系**
  - 每次成功使用 +5 点「功德值」
  - 本地记录积分与等级（如：初识表达 / 情绪译者 / 非暴力玩家 / 关键对话高手）
  - 首页底部展示功德值与升级进度

**演示模式**：未配置 `OPENAI_API_KEY` 时，接口自动返回示例数据，面试官/他人打开链接即可完整体验流程，无需配置。

## 技术栈

- **框架**：Next.js（App Router）+ TypeScript
- **UI**：自定义极简温暖样式（无 UI 库）
- **AI 调用**：默认使用 OpenAI Chat Completions API（`gpt-4.1-mini`，可按需修改）
- **部署**：推荐 Vercel 一键部署

## 本地开发

```bash
npm install
npm run dev
```

访问 `http://localhost:3000` 即可打开「好好说话」首页。

## 配置 OpenAI Key

在本地开发时：

```bash
export OPENAI_API_KEY=你的key
npm run dev
```

在 Vercel 部署时：

1. 在 Vercel 项目设置中进入 **Environment Variables**
2. 添加一条：
   - `OPENAI_API_KEY = 你的 key`
3. 重新部署

## 部署到 Vercel

1. 将本项目推送到 Git 仓库（GitHub / GitLab 等）
2. 打开 [Vercel 官网](https://vercel.com) 并使用 Git 账号登录
3. 选择 **New Project** → 导入对应仓库
4. 确认项目根目录（即包含 `package.json` 的目录）
5. 在 **Environment Variables** 填写 `OPENAI_API_KEY`
6. 点击部署，完成后即可拿到公开访问链接

## 逻辑与边界说明（已检查）

- **情绪翻译**：未输入原话时前端拦截并提示；后端缺 `text` 返回 400。
- **沟通训练**：无题目时主按钮为「生成训练题」；有题目未写答案时点击「提交打分」会提示「先写一句你会说的话」；后端对空 `trainAnswer` 评分请求返回 400。
- **再来一题**：仅请求新题目（不传 `trainAnswer`），成功后清空当前题目、答案与评语，并 +5 功德。
- **API**：无 `OPENAI_API_KEY` 时走演示数据（翻译 / 出题 / 评分均有 mock），不报错；有 Key 时正常调 OpenAI，评分输出要求【总体】【亮点】【可优化】【关联】分段便于前端展示。
- **功德与等级**：仅用 `stats.points` 计算等级与进度条；数据存 localStorage，清除浏览器数据会重置。
